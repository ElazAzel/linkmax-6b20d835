# Аудит локалей (Locale Audit)

Полный аудит файлов переводов в `src/i18n/locales/`. Эталон структуры — `en.json` (4237 листовых ключей).

## Как запустить аудит

```bash
npm run i18n:audit
```

Скрипт выводит отчёт в консоль и сохраняет его в `reports/locale-audit-YYYY-MM-DD.txt`.

## Метрики

| Метрика | Описание |
|--------|----------|
| **Пусто** | Количество ключей с пустой строкой `""` |
| **Нет ключа** | Ключи, которые есть в `en.json`, но отсутствуют в данной локали |
| **= EN** | Количество ключей, значение которых совпадает с английским (потенциальные заглушки) |
| **Заполнено %** | Доля ключей с непустым уникальным значением от общего числа ключей эталона |

## Сводка по состоянию (актуальный отчёт — в `reports/`)

- **Полностью приведены к эталону (0 пустых, 0 отсутствующих):** `en`, `ru`, `kk`, `uz` (у uz 102 отсутствующих ключа).
- **Много пустых и отсутствующих ключей:** ar, be, de, es, fr, it, ja, ko, pt, tr, uk, zh — после `i18n:fill` структура выровняется, пустые заполнятся из en.
- **Заглушки (= EN):** во всех неанглийских локалях часть ключей совпадает с en — их стоит заменить на перевод.

## Рекомендации

1. **Пустые ключи:** выполнить `npm run i18n:fill` — недостающие ключи и пустые значения будут взяты из `en.json`.
2. **Отсутствующие ключи:** тот же `i18n:fill` добавит ключи из эталона во все локали.
3. **Заглушки (= EN):** заменить совпадающие с английским значения на перевод на соответствующий язык.
4. **Лишние ключи (20 в части локалей):** ключи есть в локали, но отсутствуют в `en.json` — можно оставить или удалить после проверки.

## Связанные скрипты

- `npm run i18n:check-coverage` — быстрый подсчёт пустых строк по локалям.
- `npm run i18n:fill` — синхронизация структуры с `en.json` и заполнение пустых значений из английского.
- `npm run i18n:audit` — полный аудит (структура, пустые, отсутствующие, заглушки).
