# lnkmx — Ультимативный гид по платформе (v2026.02)

Платформа lnkmx — это не просто конструктор персональных страниц. Это полноценная **Business Operating System (Solo OS)**, спроектированная для экономики независимых специалистов (Solo-Economy). Мы объединили создание сайтов, управление клиентами, аналитику и финансовые инструменты в единую мобильную экосистему.

---

## 1. Стратегическое видение

**Миссия**: Стереть границы между маркетингом и операционным управлением для микро-бизнеса. Мы устраняем "налог на инструменты" (необходимость платить за 5+ разных сервисов), предоставляя унифицированную инфраструктуру «всё-в-одном» за 15 минут.

---

## 2. Основные модули платформы

### 2.1 AI Page Builder (Конструктор страниц)

Ядро системы, использующее премиальную дизайн-систему **"Liquid Glass"**.

- **Блоки (28+ типов)**: От простых текстовых и ссылок до сложных (Каталоги, Бронирование, Ивенты, Скретч-карты).
- **Дизайн**: Использование глубоких теней, стекломорфизма и микро-анимаций. Каждый сайт выглядит как продукт дорого агентства.
- **AI-Onboarding**: Генерация структуры и контента страницы на основе ниши пользователя через Gemini AI. Галлюцинации исключены благодаря жестким промптам.
- **Мультиязычность**: Полная синхронизация контента на RU, EN, KK, UZ.

### 2.2 Mini-CRM и Управление лидами

Автоматизированная воронка продаж для каждого пользователя.

- **Leads Inbox**: Единая точка сбора заявок из всех форм на странице.
- **Pipeline**: Статусы сделок (New -> Contacted -> Qualified -> Won/Lost).
- **Уведомления**: Мгновенные алерты в Telegram-бот о новых записях или лидах.
- **Automations**: Почасовые процессы для фоновой обработки правил (например, напоминания о записях).

### 2.3 Business Zones (Мультиарендность)

Инструментарий для команд и малого бизнеса.

- **Workspaces**: Полная изоляция командных данных.
- **RBAC**: Ролевая модель (Владелец, Админ, Участник, Наблюдатель).
- **Team CRM**: Совместная работа над сделками, распределение задач и общий инбокс сообщений.

### 2.4 Advanced Analytics & Pixel Proxy

Решение для мира без cookie.

- **Server-side Analytics**: Проксирование запросов к Facebook CAPI, TikTok Events API и GA4 через наши Edge Functions.
- **Точность данных**: Повышение точности трекинга на 30% за счет обхода блокировщиков рекламы на стороне сервера.
- **Heatmaps**: Визуализация кликов по блокам для оптимизации конверсии.

### 2.5 Fintech Core

Финансовый фундамент платформы.

- **Ledger System**: Жесткая бухгалтерская запись всех транзакций.
- **Wallets**: Мультивалютные кошельки для пользователей.
- **GMV Tracking**: Мониторинг оборота через инвойсы и продажи товаров/услуг.

## 3. Основные пользовательские сценарии (User Journeys)

### 3.1 Для создателя (Creator)

1. **Онбординг**: Регистрация -> AI-опросник -> Мгновенная генерация страницы.
2. **Кастомизация**: Настройка блоков через `UnifiedBlockEditor`, выбор темы, настройка SEO и Пикселей.
3. **Публикация**: Привязка кастомного домена (CNAME) и нажатие "Publish" (создается версионный снепшот).
4. **Управление**: Получение уведомлений о лидах в Telegram -> работа в CRM -> анализ Insights.

### 3.2 Для посетителя (Visitor)

1. **Просмотр**: Быстрая загрузка страницы (оптимизировано LCP).
2. **Конверсия**: Запись на прием (Booking), регистрация на ивент или покупка товара через каталог.
3. **Трекинг**: Действия анонимно трекаются через Pixel Proxy для ретаргетинга владельцем.

## 4. Детальный технический разбор

### 4.1 Frontend Компоненты

- **Block Layout**: `BlockRenderer.tsx` — динамический диспетчер, использующий `lazy()` для подгрузки только необходимых блоков.
- **Editor Core**: `useBlockEditor.tsx` на базе **Zustand** обеспечивает реактивность без лагов при перетаскивании блоков.
- **Micro-animations**: Использование Framer Motion и кастомных CSS-переменных для эффектов "загрузки" и "свечения".

### 4.2 Backend & Data

- **Edge Functions**: 28 изолированных функций. Например, `pixel-proxy` обрабатывает до 1000 событий в секунду, не нагружая основную БД.
- **Realtime**: Использование Supabase Realtime для мгновенного обновления сообщений в командном инбоксе.
- **Fintech Ledger**: Реализация через транзакционные SQL-функции с защитой от двойного списания.

---

## 5. Инфраструктура

- **Cloudflare Workers**: Кэширование на границе (Edge Caching) и детекция ботов.
- **Image Optimization**: Проксирование всех изображений через Supabase Storage Transformation для выдачи WebP/AVIF.
- **Backup**: Ежедневные бэкапы Postgres и версионность страниц в таблице `page_snapshots`.

---

## 6. Техническая архитектура

### 6.1 Frontend Stack

- **Framework**: React 18 + Vite (SPA).
- **State Management**: **Zustand** (глобальное состояние редактора) + **React Query** (серверные данные).
- **Styling**: Tailwind CSS + shadcn/ui.
- **Performance**: PWA, Lazy Loading компонентов, оптимистичные обновления интерфейса.

### 6.2 Backend & Cloud Infrastructure

- **Core**: Supabase (PostgreSQL, Auth, Storage).
- **Edge Compute**: 28+ функций на Deno (Supabase Edge Functions).
- **SEO/SSR**: Гибридный подход. Cloudflare Workers детектируют ботов и отдают предрендеренный HTML через специальную Edge-функцию `seo-ssr`.
- **Database Security**: Мощная система **RLS (Row Level Security)** гарантирует, что пользователь никогда не получит доступ к чужим данным.

### 6.3 AI Integration

- **LLM**: Google Gemini для генерации текстов, перевода и анализа данных.
- **Automation**: Интеграция с Telegram Bot API для бесшовного управления бизнесом "на ходу".

## 8. Стратегические методологии и аналитика

Для успешного развития lnkmx как продукта и бизнеса мы используем набор проверенных методологий, которые помогают нам лучше понимать пользователя и принимать data-driven решения.

### 8.1 Customer Journey Map (CJM) — Карта пути клиента

Это визуализация опыта пользователя от момента возникновения потребности до лояльного использования.

- **Осведомленность**: Пользователь видит рекламу в Instagram или ссылку "Powered by lnkmx" у другого эксперта. *Страх*: "Сложно настраивать?". *Мотив*: Сделать красиво как у других.
- **Рассмотрение**: Попадание на LandingV5. *Ожидание*: Быстрый старт.
- **Первое касание**: Регистрация и AI-онбординг. *Цель*: Увидеть свою страницу через 2 минуты.
- **Использование**: Настройка блоков, публикация.
- **Покупка (Upgrade)**: Столкновение с лимитами Free-типа или потребность в CRM.
- **Лояльность**: Использование Business Zones для всей команды, привязка домена.

### 8.2 Customer Development (CustDev)

Методология проверки гипотез через интервью с реальными пользователями.

- **Гипотеза 1**: Пользователям не хватает Kanban-доски в CRM.
- **Проверка**: Интервью с 20 beauty-мастерами. Результат: Проблема не в доске, а в напоминаниях клиентам.
- **Инсайт**: Фокус смещен с визуализации на автоматизацию (Edge Functions для уведомлений).

### 8.3 Jobs To Be Done (JTBD)

Мы продаем не "конструктор сайтов", а "выполненную работу".

- *Job*: "Когда я начинаю новый курс, я хочу быстро собрать страницу регистрации, чтобы не тратить 2 дня на техническую настройку и сразу начать лить трафик".
- *Инсайт*: Фокус lnkmx на скорости и преднастроенных блоках (Event Block) важнее, чем глубокая кастомизация CSS.

### 8.4 User Story Mapping (USM)

Структурирование бэклога в виде карты пользовательского опыта.

- **Хребет (Backbone)**: Регистрация -> Создание контента -> Управление продажами -> Аналитика.
- **Наполнение**: Под каждой темой располагаются задачи (Stories), приоритизированные по релизам (MVP, V2, V3).

### 8.5 Unit Economics (Юнит-экономика)

Расчет прибыльности одного пользователя (юнита).

- **LTV (Lifetime Value)**: Сколько денег приносит пользователь за всё время.
- **CAC (Customer Acquisition Cost)**: Стоимость привлечения одного платного пользователя.
- **Цель**: LTV > 3 * CAC.

### 8.6 Pirate Metrics (AARRR)

Воронка роста платформы.

- **Acquisition (Привлечение)**: Виральный эффект от ссылок пользователей.
- **Activation (Активация)**: Публикация первой страницы.
- **Retention (Удержание)**: Ежедневная проверка CRM и уведомлений.
- **Referral (Рефералы)**: Система Linkkon токенов за приглашения.
- **Revenue (Доход)**: Конверсия в Pro Tier и White-label.

### 8.7 North Star Metric (Метрика "Полярной звезды")

Основной показатель ценности продукта для пользователей.

- **Для lnkmx**: Количество "Активных Лидов" (Leads Capture), прошедших через CRM пользователей за месяц. Если лиды идут — платформа приносит пользу.

### 8.8 SWOT-анализ

- **Strengths**: Дизайн Liquid Glass, скорость работы, встроенная CRM.
- **Weaknesses**: Отсутствие нативного моб. приложения (в разработке), зависимость от Supabase.
- **Opportunities**: Рост соло-экономики в СНГ и ЮВА, внедрение Fintech платежей.
- **Threats**: Копирование дизайна конкурентами, агрессивный демпинг Linktree.

### 8.9 HADI-циклы

Метод быстрой проверки гипотез разработки.

- **Hypothesis**: Если добавить блок "Скретч-карта", конверсия вырастет на 5%.
- **Action**: Реализация блока за 1 спринт.
- **Data**: Анализ кликов через Insights.
- **Insights**: Подтверждение или опровержение.

### 8.10 RICE Scoring

Приоритизация задач в бэклоге.

- **Reach**: Сколько юзеров затронет фича.
- **Impact**: Насколько сильно поможет.
- **Confidence**: Уверенность в оценке.
- **Effort**: Затраты времени.

### 8.11 Lean Canvas

Бизнес-модель lnkmx на одной странице (Проблема, Решение, Уникальность, Сегменты).

### 8.2 Cohort Analysis (Когортный анализ)

Анализ поведения групп пользователей, пришедших в разное время. Помогает понять влияние новых фич на удержание (Retention).

### 8.13 Marketing Mix (7P)

- **Product**: Business OS.
- **Price**: $7/mo.
- **Place**: Global Web / PWA.
- **Promotion**: Influencers + Viral Loop.
- **People**: Соло-пренеры.
- **Process**: Мгновенный онбординг.
- **Physical Evidence**: Премиальный дизайн страниц.

### 8.14 OKRs (Objectives and Key Results)

- **Objective**: Стать №1 платформой для экспертов в Центральной Азии.
- **Key Result 1**: 50,000 активных страниц.
- **Key Result 2**: Конверсия в Pro > 5%.
- **Key Result 3**: NPS (индекс лояльности) > 70.
