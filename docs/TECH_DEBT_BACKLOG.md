# Технический долг и Критические уязвимости (Backlog)

> **Источник данных:** Стратегический аудит платформы от 18 февраля 2026 г.
> **Цель:** Устранение критических барьеров перед масштабированием и фандрейзингом.

## Критические задачи (P0 — Remediation Required)

### 1. Progressive Web App (PWA)
- **Проблема:** Платформа позиционируется как mobile-first, но установка на смартфон невозможна.
- **Статус:** Не функционирует (Health Score: 3/10).
- **Что сделать:** 
    - [ ] Создать валидный `manifest.json` в корне.
    - [ ] Реализовать `service-worker` (`sw.js`).
    - [ ] Обеспечить работу базового функционала в офлайн-режиме.

### 2. SEO & Organic Discovery
- **Проблема:** Файл `sitemap.ts` отключен/переименован в `.bak`, индексация страниц пользователей заблокирована.
- **Статус:** Искусственно заблокировано (Health Score: 7/10).
- **Что сделать:** 
    - [ ] Восстановить работу `sitemap.ts`.
    - [ ] Проверить индексацию поисковыми роботами.

### 3. Monetization & Payments (Preparation)
- **Проблема:** Отсутствует интеграция с реальными фиатными платежами (Kaspi, Stripe). Текущая система токенов "имитационная".
- **Статус:** Стадия имитации (Health Score: 5/10).
- **Что сделать:** 
    - [ ] Провести R&D по Kaspi API (или Robokassa).
    - [ ] Спроектировать слой транзакций для реальных денег.

## Высокий приоритет (P1 — UX & Stability)

### 4. Authentication (OAuth Redirects)
- **Проблема:** Параметр `returnTo` игнорируется при входе через Google/Apple, прерывая deep-linking.
- **Статус:** Нарушение UX (Health Score: 8/10).
- **Что сделать:** 
    - [ ] Исправить логику обработки редиректов в `src/pages/Auth.tsx` и Edge Functions.

### 5. System Stability (Block Registry Drift)
- **Проблема:** Рассинхронизация `block-registry.ts`. Блок `search` есть в лимитах, но нет в реестре. Тяжелые шаблоны захардкожены (99KB+).
- **Статус:** Риск Runtime ошибок (Health Score: 9/10).
- **Что сделать:** 
    - [ ] Синхронизировать реестр блоков.
    - [ ] Вынести тяжелые шаблоны из кодовой базы в БД или CDN.

### 6. Telegram Bot Architecture
- **Проблема:** Отсутствуют команды `/stats` и `/publish` для админов. Настройки языка сбрасываются при "холодном старте" Edge Functions.
- **Статус:** Функционально ограничен (Health Score: 7/10).
- **Что сделать:** 
    - [ ] Перенести настройки бота в БД (вместо in-memory).
    - [ ] Реализовать недостающие команды.
