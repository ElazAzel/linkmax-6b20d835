

# План развития платформы lnkmx — Март-Апрель 2026

## Текущее состояние (по результатам аудита кода)

**Что уже работает хорошо:**
- 28+ блоков конструктора страниц, SEO/SSR, i18n (4 языка)
- Бизнес-зоны: CRM, Сделки, Контакты, Задачи, Инвойсы, Документы, Продукты, Ивенты, Автоматизации
- Аналитика с воронкой, графиками, экспортом в Excel
- Лиды с фильтрацией, статусами, быстрыми действиями (WhatsApp/Telegram/Call)
- PWA-оболочка, Capacitor для iOS/Android
- Локализация на 4 языках (RU/EN/KK/UZ)

**Критические разрывы между стратегией и кодом:**

| Проблема | Суть |
|---|---|
| **Тарифная модель устарела** | Стратегия требует Identity/Starter/Pro/Business, код знает только free/pro/business |
| **Комиссия 5% вместо 7%** | `fintech.ts` использует `DEFAULT_TAKE_RATE = 0.05`, стратегия требует 7% для Starter |
| **MonetizeScreen не знает про Starter** | Показывает только Free/Pro, нет тарифа "0₸ + 7% комиссия" |
| **Биллинг — заглушки** | AccountSettings: `{/* TODO: Open billing history */}` и `{/* TODO: Open password change */}` |
| **PaymentService — скелет** | `create-payment-session` Edge Function существует, но нет реального процессинга |
| **Нет Kaspi интеграции** | `kaspi-service.ts` существует, но не интегрирован в UI сделок/инвойсов |

## План на ближайшие 6 недель

### Неделя 1 (9-15 марта): Тарифная модель

**Цель:** Привести код в соответствие со стратегией Identity/Starter/Pro/Business.

- Обновить `PremiumTier` type: `'identity' | 'starter' | 'pro' | 'business'`
- Обновить `useFreemiumLimits.ts`: добавить лимиты Starter (все блоки, CRM, но с комиссией 7%)
- Обновить `checkPremiumStatus` в `services/user.ts` для поддержки новых тиров
- Обновить `MonetizeScreen.tsx`: показывать 4 тарифа с ценами в KZT
- Обновить `fintech.ts`: динамическая комиссия (Starter=7%, Pro=1%, Business=0%)
- Обновить pricing page для соответствия

### Неделя 2 (16-22 марта): Платежи и биллинг

**Цель:** Рабочий платежный флоу Robokassa + история биллинга.

- Доработать Edge Function `create-payment-session` для реального создания заказа
- Реализовать `robokassa-webhook` для обработки успешных оплат
- Создать таблицу `billing_history` и UI для отображения в AccountSettings
- Реализовать смену пароля в AccountSettings (убрать TODO)
- Интегрировать Kaspi QR в карточку сделки (базовый уровень)

### Неделя 3 (23-29 марта): Отчеты и бизнес-аналитика

**Цель:** ZoneAnalytics как полноценный операторский дашборд.

- Добавить P&L отчет (доходы vs расходы по инвойсам)
- Добавить воронку конверсии сделок по стадиям с трендами за период
- Добавить метрики команды (задачи по исполнителям, скорость закрытия)
- PDF-экспорт отчетов
- Добавить метрики биллинга: конверсия Starter→Pro, GMV

### Неделя 4 (30 марта - 5 апреля): Мобильный UX

**Цель:** Все экраны удобны для работы одной рукой.

- Аудит touch targets: минимум 44px на все кнопки/ссылки
- Убрать `text-[10px]`, `text-[11px]` из всех zone-экранов
- Добавить error boundaries на каждый zone-экран
- Улучшить empty states и loading skeletons
- Responsive QA: Dashboard, Zones, Invoices, Bookings

### Неделя 5 (6-12 апреля): PWA V2 + Offline

**Цель:** Оффлайн-чтение контактов, задач, инвойсов.

- Расширить service worker: кеширование API-ответов для contacts, tasks, invoices
- Индикатор свежести данных (последняя синхронизация)
- Disable мутаций в оффлайн-режиме с понятным сообщением
- Валидация install/update для PWA и Capacitor

### Неделя 6 (13-19 апреля): Релиз

**Цель:** Go/No-Go для запуска Starter.

- Регрессия: auth, создание страницы, CRM workflow, оплата, аналитика
- Юридические тексты: Terms, Privacy, Payment Terms
- Мониторинг: алерты на ошибки оплат и RLS-нарушения
- KPI-дашборд: Starter adoption rate, Pro conversion, GMV
