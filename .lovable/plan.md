

# Анализ и план оптимизации i18n

## Текущее состояние

**Архитектура:** i18next + react-i18next, 16 языков, 3 основных (ru, en, kk) загружаются eagerly, 13 — lazy. Каждый locale-файл ~6720 строк (~5400+ ключей).

**Fallback-цепочка:** kk → en, uz → en, остальные → en. Русский НЕ в fallback-цепочке (но является основным языком продукта).

## Найденные проблемы

### 1. Захардкоженные русские строки в компонентах (11 файлов, 89+ мест)
Строки на русском вбиты прямо в JSX, минуя `t()`. Примеры:
- `Install.tsx` — инструкции по установке PWA
- `BlockHint.tsx` — подсказки к блокам
- `AIBuilderWizard.tsx` — плейсхолдеры и хинты
- `AdminUsersTab.tsx` — placeholder "Подписка"
- `CatalogBlockEditor.tsx`, `AIInsightsPanel.tsx` и др.

При переключении языка эти строки остаются на русском.

### 2. Неиспользуемые pricing fragment файлы (16 шт)
Файлы `pricing_*_fragment.json` нигде не импортируются — мёртвый код, увеличивают размер репозитория.

### 3. Fallback-цепочка не оптимальна
- `default: ['en']` — но продукт ориентирован на русскоязычных. Если ключ отсутствует в kk/uz, пользователь видит английский, а не русский.
- Лучше: `kk → ru → en`, `uz → ru → en`.

### 4. Странный корневой ключ `"2": { "5 MB": "5 МБ" }`
Во всех локалях есть бессмысленный ключ `"2"."5 MB"`. Вероятно, артефакт.

### 5. Русские fallback-строки в `t()` вызовах
Многие `t('key', 'Русский текст')` используют русский как дефолт. Если ключ отсутствует, пользователь на английском увидит русский текст. Fallback должен быть на английском или убран вовсе.

## План оптимизации

### Шаг 1. Вынести захардкоженные строки в i18n (11 файлов)
Заменить все `"Русский текст"` в JSX на `t('ключ')` и добавить ключи во все 4 основные локали (ru, en, kk, uz).

### Шаг 2. Исправить fallback-цепочку
```ts
fallbackLng: {
  'kk': ['ru', 'en'],
  'uz': ['ru', 'en'],
  'uk': ['ru', 'en'],
  'be': ['ru', 'en'],
  'default': ['en']
}
```
Для постсоветских языков fallback сначала на русский, потом на английский.

### Шаг 3. Исправить русские дефолты в `t()` вызовах
Заменить `t('key', 'Русский текст')` на `t('key')` — пусть i18next использует fallback-цепочку вместо хардкода.

### Шаг 4. Удалить мёртвые pricing fragment файлы
Удалить 16 файлов `pricing_*_fragment.json` из `src/i18n/locales/`.

### Шаг 5. Удалить артефактный ключ `"2"."5 MB"`
Убрать из всех locale-файлов.

### Оценка объёма
- Шаги 1 и 3 — основной объём работы (~11 компонентов + добавление ключей в 4 JSON файла)
- Шаги 2, 4, 5 — быстрые правки
