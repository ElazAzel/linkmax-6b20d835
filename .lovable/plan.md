

# Demo Asset Pack for Founder-Led GTM

## Verdict

The outreach playbook exists but has no tangible proof assets to attach. A cold DM with a link to a real, polished demo page converts 2-3x better than a DM with just text. This is a documentation/content deliverable, not a code change.

## What to Build

**One file: `docs/gtm/DEMO_ASSET_PACK.md`** — a comprehensive operational document containing:

### 1. Three Demo Page Content Specs

Each spec includes ready-to-paste content for building pages on the platform:

**Demo #1 — `demo_nails` (Nail master "Айгерим")**
- ICP: Independent nail tech, Almaty, Instagram-first
- Full content: bio, 6 services with prices (3000-12000₸), booking settings, trust copy, FAQ
- Block order: Profile → Services/Prices → Booking → Reviews → FAQ → WhatsApp fallback

**Demo #2 — `demo_lashes` (Lash/brow master "Динара")**
- ICP: Lash extension + brow artist, Astana
- Full content: bio, 5 services (5000-15000₸), booking, proof block
- Focus: before/after positioning, repeat booking emphasis

**Demo #3 — `demo_barber` (Barber "Арман")**
- ICP: Male grooming, appointment-based
- Full content: bio, 4 services (2500-8000₸), booking
- Focus: speed/efficiency messaging, no-frills

### 2. Reusable Page Structure Template
Optimal block order that works across all beauty niches.

### 3. DM Asset Sequence
Exact mapping: which asset goes with which outreach message from the playbook.

### 4. Screenshot Plan
6 specific screenshots to capture from the demo pages, with crop instructions and captions.

### 5. Video Script
15-second screen recording flow: Instagram bio → tap link → see services → pick slot → booking confirmed.

### 6. Case Study Template
Reusable before/setup/after/result structure for first 5 users.

### 7. Copy Pack
12 ready-to-use micro-copy lines for captions, CTAs, and proof.

## Implementation

- **Single task**: Create `docs/gtm/DEMO_ASSET_PACK.md` with all content above
- No code changes to the app
- The founder then uses these specs to build 3 real pages on the platform using existing demo accounts (demoaccount1, demoaccount2 can be repurposed, or new ones created)

## What NOT to Do
- No code-generated demo pages (they should be real pages built on the platform for authenticity)
- No new routes or components
- No changes to existing pages
