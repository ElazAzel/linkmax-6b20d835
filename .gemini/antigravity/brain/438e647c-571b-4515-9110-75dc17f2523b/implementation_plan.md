# Стратегия развития и "допинги" LinkMAX (Phase 45: Performance & Modernization)

Изучив текущий стек проекта (Vite 6, React 18.3, Supabase, Tailwind, Capacitor 8, Zustand, React Query 5) и ваши цели (B2B SaaS, CRM, микробизнес), я подобрал список современных, безопасных и актуальных для **2026 года** инструментов. 

Главное правило — **Zero-Breaking Changes**. Мы добавляем это слоями, не ломая текущую бизнес-логику платформы.

## 1. Frontend Core: Производительность и Реактивность

### React Compiler (Babel Plugin)
В текущем коде много `useCallback` и `useMemo` (например, в хуках аналитики и CRM). В 2025-2026 стандартом стал **React Compiler**, который работает поверх React 18+ и автоматически оптимизирует ререндеры, избавляя от необходимости писать ручные мемоизации.
- **Пакет:** `babel-plugin-react-compiler` + `@vitejs/plugin-react`
- **Риск:** Минимальный (ставится в режиме report/safe-mode).

### Виртуализация DOM для CRM (Million.js / TanStack Virtual)
Так как CRM доски (Kanban) и таблицы заказов будут расти, стандартный рендер DOM начнет тормозить.
- **Решение:** Добавление `@tanstack/react-virtual` для бесконечных списков в дашборде/ленте заявок. Не меняет логику, только оптимизирует DOM.

## 2. Mobile Native Feel (Capacitor 8 Enhancements)

Для того чтобы приложение, собранное через Capacitor, ощущалось "нативным" за $1М, нам нужны следующие плагины (допинги для UX):
- **Tactile Feedback (Haptics):** Добавить `@capacitor/haptics`. При перетаскивании сделки в Kanban, кликах на кнопки переходов, открытии Dialog — легкая вибрация экрана. Это радикально меняет восприятие интерфейса.
- **Keyboard Handling:** `@capacitor/keyboard` для плавного скролла контента над клавиатурой на iOS/Android при вводе в чат-бот или формы.
- **Local-first (Offline Ready):** Интеграция `@tanstack/react-query-persist-client` + IndexedDB. CRM должна грузиться мгновенно из кэша телефона до ответа сервера.

## 3. Product Analytics & Telemetry (Из Roadmap)

В предыдущих чеклистах упоминалась потребность в PostHog и мониторинге.
- **PostHog (`posthog-js`):** Современный стэк аналитики. Заменит самописные Event Tracking (heatmap, session replay, A/B тестирование блоков без необходимости писать SQL запросы).
- **Синхронизация Sentry:** Пакет `@sentry/react` уже стоит, но нужно активировать **Session Replay** для отлова скрытых багов (особенно пустых экранов у пользователей с проблемным интернетом).

## 4. DX и Quality Assurance (Удержание качества)

- **Lefthook / Husky:** У вас крутые скрипты в `package.json` (`quality:gate`, `analyze:cycles`), но нет Git Hooks. Разработчик (или агент) может запушить невалидный код. Внедрение `lefthook` (запускает Prettier/ESLint на измененных файлах перед коммитом) спасет от регрессий.
- **Nuqs (Type-Safe URL State):** Вместо `useState` для фильтров CRM, в 2026 году используют `nuqs`. Это синхронизирует стейт с URL. *Плюс:* можно отправить ссылку на отфильтрованную доску коллеге, и она откроется с теми же фильтрами.
