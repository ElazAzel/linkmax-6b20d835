# Правила фронтенда (React)

## 1. Структура компонентов

- Используйте `src/components/` для общих компонентов.
- Используйте `src/pages/` для компонентов страниц.
- Используйте `src/hooks/` для кастомных хуков.

## 2. Управление состоянием

- Предпочитайте `useState` для локального состояния.
- Используйте **React Query** для серверного состояния.
- Используйте Context только для небольших глобальных нужд (Auth, Theme).

## 3. Стилизация

- Используйте компоненты **Shadcn UI** из `@/components/ui/`.
- Используйте **Tailwind CSS** для верстки и кастомных стилей.
- Избегайте "чистых" CSS файлов, если это не строго необходимо.

## 4. Производительность

- Мемоизируйте тяжелые вычисления через `useMemo`.
- Мемоизируйте колбэки через `useCallback` при передаче в дочерние компоненты.
- Используйте `React.lazy` для ленивой загрузки роутов.

## 5. Тестирование

- Используйте `vitest` + `react-testing-library` для модульных тестов.
- Тестируйте поведение пользователя, а не детали реализации.

## 6. Правила дизайна

- **Блоки Divider и Social**: НЕ должны иметь границ (frames) или рамок. Они должны быть "чистыми" и без рамок.
