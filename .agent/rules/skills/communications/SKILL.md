# Communications Skill

Управление коммуникациями платформы: уведомления пользователям, внутренние апдейты команды, email-рассылки.

## Когда использовать

- Формирование 3P-апдейта (Progress/Plans/Problems)
- Настройка Telegram-уведомлений о лидах и бронированиях
- Написание ответов на FAQ пользователей
- Создание новостных рассылок и changelog'ов
- Интеграция с Gmail для отправки писем

## Воркфлоу

### 1. 3P-апдейт (Progress/Plans/Problems)

Формировать по запросу или каждую пятницу:

```markdown
## Progress
- [ ] Что сделано за неделю (с ссылками на PR/коммиты)

## Plans
- [ ] План на следующую неделю (приоритеты)

## Problems
- [ ] Блокеры и риски (с предложениями решений)
```

**Источники данных:** `git log --oneline --since="7 days ago"`, свежие issues в GitHub.

### 2. Telegram-уведомления

При событии (новый лид, бронирование, оплата):
1. Определить шаблон уведомления в `src/telegram/templates/`
2. Сформировать сообщение:
   ```
   🚀 Новый лид!
   Имя: {name}
   Услуга: {service}
   Контакт: {contact}
   ```
3. Отправить через Edge Function `telegram-notify`
4. Залогировать в `notification_log`

**Ключевые файлы:**
- `supabase/functions/telegram-notify/index.ts`
- `src/telegram/` — клиент и шаблоны
- `src/repositories/notification.repository.ts`

### 3. FAQ / База знаний

При запросе пользователя:
1. Поискать в `docs/` или `src/i18n/` готовый ответ
2. Если ответа нет — сформировать новый с ссылкой на документацию
3. Добавить в `docs/operations/FAQ.md`

### 4. Email через Gmail

По необходимости (интеграция через Supabase Edge Function):
- Отправка welcome-письма при регистрации
- Уведомления об оплате
- Сброс пароля (штатный Supabase Auth)

## Связанные модули

- `business-zone` — триггеры уведомлений по лидам
- `content-creation` — контент для email-рассылок
- `changelog` — новости релизов для рассылки
